# Wallet Briefs and Feedback

Clearer wallet-specific beta briefs reduce support load and make feedback easier to act on.

## Why the briefs changed

We split generic onboarding copy into shorter, wallet-aware pages so each beta user sees the exact setup path they need.

## Feedback routing

Bug reports, wallet questions, and launch feedback should land in different places. That keeps the team from mixing product issues with onboarding confusion.

## Next step

Future update notes can document which wallet flows changed and which support patterns improved after each release.
