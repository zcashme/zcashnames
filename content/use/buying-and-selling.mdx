# Buying & Selling Names

ZNS has a non-custodial marketplace. Anyone who owns a name can list it for sale. A sale is a two-transaction protocol — the buyer pays the seller directly via transparent ZEC, the indexer enforces the rest.

## How it works

1. **Seller lists** with a transparent address (`pay_taddr`) where they want to receive payment.
2. **Buyer claims** the listing with a signed shielded memo — this locks the listing for ~2 hours.
3. **Buyer pays** the listing price directly to the seller's `pay_taddr` via transparent transaction.
4. **Indexer detects** the payment and finalizes the name transfer to the buyer.

## Listing a name for sale

1. Search for your registered name and click **List for Sale**.
2. Complete [OTP verification](/docs/use/otp-verification) to prove you control the address the name is registered to.
3. Enter your price in ZEC (or zats).
4. Provide a transparent Zcash address (`pay_taddr`) where you want to receive payment.
5. Send the transaction.

Memo format: `ZNS:LIST:{name}:{price}:{pay_taddr}:{nonce}:{sig}`

The note value must include a **0.01 ZEC listing commission** — this is non-refundable and covers the lock slot for any buyer.

### Changing the price

List again with a new price. The indexer uses `INSERT OR REPLACE` on the listings table, so the new row overwrites the old one. The nonce must advance each time.

## Buying a listed name

The buy flow has two steps — you must complete both.

### Step 1: Claim the listing

1. Search for a name that shows as **For Sale**.
2. Enter your Zcash unified address.
3. The app generates a ZIP-321 payment URI for the listing price with a memo: `ZNS:BUY:{name}:{buyer_ua}:{price}:{sig}`.
4. The note value includes a **0.0001 ZEC dust commission** — this is non-refundable.

The BUY claim locks the listing for approximately **2 hours** (144 blocks). If you do not complete Step 2 within that window, the lock expires and the name returns to the listing.

### Step 2: Pay the seller

After your BUY claim confirms and the indexer registers the pending buy:

1. Send exactly the listing price in ZEC as a **transparent transaction** to the seller's `pay_taddr` (shown in the listing).
2. Wait for the payment to confirm.

The indexer monitors for a transparent output to the seller's `pay_taddr` that meets or exceeds the listing price. The first qualifying payment wins. After confirmation, the name transfers to your address and the listing is removed.

## Price limits

| | Zats | ZEC |
|-|------|-----|
| Protocol minimum | 1 | 0.00000001 |
| Web UI minimum | 100,000 | 0.001 |
| Protocol maximum | 2,100,000,000,000,000 | 21,000,000 (Zcash supply cap) |

The web UI is deliberately more conservative than the protocol.

## Finding listings

The easiest way to browse the marketplace is the [Explorer](/explorer) — switch to the **For Sale** tab to see every active listing across networks, with the seller and price in one view.

Programmatically, call `listings` to get active listings with pagination support. Results are ordered newest first:

```bash
# Get the first 50 listings (default page)
curl -sX POST https://light.zcash.me/zns-testnet \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"listings","params":{"limit":50,"offset":0}}' | jq
```

You can request up to 500 listings per page:

```bash
# Get listings 101-200 (second page with custom size)
curl -sX POST https://light.zcash.me/zns-testnet \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"listings","params":{"limit":100,"offset":100}}' | jq
```

The response includes each listing's signature so clients can verify independently. See [Resolution](/docs/learn/resolution) for the signature semantics.

**Response format:**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "listings": [...],
    "total": 123
  }
}
```

The `total` field indicates the total number of active listings available.

## Delisting

Remove your name from the marketplace while keeping the registration:

1. Search for your listed name and click **Delist**.
2. Complete OTP verification.
3. Send the transaction.

Memo: `ZNS:DELIST:{name}:{nonce}:{sig}`

**Delisting is blocked while a pending buy is active.** If a buyer has claimed the listing and the lock has not expired, the indexer will reject the DELIST until the lock clears (~2 hours) or the sale finalizes.

## Commission model

| Action | Amount | Who pays | Refundable? |
|--------|--------|----------|-------------|
| LIST | 0.01 ZEC | Seller | No — burned even if the listing never sells |
| BUY | 0.0001 ZEC | Buyer | No — burned even if buyer never pays |

The commissions prevent spam and fund the lock slots. Both are prepaid as note values on the respective memos.

## Gotchas

- **BUY is a two-step process.** You must claim first (shielded memo), then pay second (transparent tx to the seller's `pay_taddr`). Skipping steps means the indexer won't finalize the sale.
- **The payment must go to the seller's `pay_taddr`** — not the registry, not the indexer. The indexer watches for transparent outputs to that address.
- **Buying does not refund overpayment.** If the listing is 1 ZEC and you send 1.5, you pay 1.5. The app should always use the exact listing price.
- **If the lock expires before you pay, you must start over.** The BUY claim locks the listing for ~2 hours (144 blocks). After that, the name is available to any buyer again.
- **Overlapping BUY claims in the same block** — the first processed wins. The indexer enforces first-claim-wins via a unique constraint on the listing key.
- **There is no escrow.** Funds go directly from buyer to seller. The indexer only monitors for the payment; it never holds funds.
