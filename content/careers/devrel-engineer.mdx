# Developer Relations Engineer, SDK Integrations

## Purpose

Own successful SDK adoption from first contact to production integration. Reduce founder load by handling technical partner support, docs, examples, and integration feedback, and turn partner pain into product improvements.

## Responsibilities

- Build and maintain SDK examples, starter apps, and integration guides.
- Support partners through sandbox, testnet, mainnet, launch, and post-launch.
- Debug issues across SDK, wallet, resolver, and app flows.
- Write API references, tutorials, changelogs, migration guides, and release notes.
- Maintain sample repos and developer-facing demos.
- Triage issues and file actionable engineering tickets.

## Required

- Experience shipping production software.
- Strong TypeScript or JavaScript.
- Experience integrating SDKs, APIs, wallets, or developer tools.
- Clear technical writing.
- Ability to debug across frontend, backend, and network boundaries.
- Comfortable communicating with external developers.

## Strong Plus

- Zcash, privacy, crypto naming, identity, or payments experience.
- Rust, React, Node.js, WASM, or mobile wallet experience.
- DevRel, solutions engineering, or SDK engineering background.

## Success

- Time from partner intro to working integration decreases.
- Docs and examples stay current with releases.
- Partners can integrate without private explanations.
- Developer feedback turns into product improvements.
