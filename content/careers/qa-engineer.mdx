# QA / Release Engineer

## Purpose

Own release quality before, during, and after production launch. Build a repeatable release process and catch regressions across wallet, SDK, resolver, app, and docs flows.

## Responsibilities

- Maintain the release checklist and regression test plan.
- Test SDK, wallet, resolver, app, and docs flows before release.
- Validate staging, testnet, and mainnet behavior.
- Perform post-release smoke tests.
- Reproduce bugs and write actionable reports.
- Add lightweight automation where it reduces risk.

## Required

- Experience testing shipped software.
- Comfortable with APIs, SDKs, web apps, and release artifacts.
- Strong bug-report discipline with logs, steps, and screenshots.
- Familiarity with GitHub, CI, browser devtools, and CLI tools.
- Good judgment about blocker vs regression vs edge case.

## Strong Plus

- Crypto wallet, payments, privacy, or blockchain testing experience.
- TypeScript and Playwright, Cypress, Jest, or Vitest experience.
- Experience testing SDKs or developer tools.

## Success

- Launch has a release checklist and regression matrix.
- Core flows are tested before launch.
- Post-release verification happens every release.
- Founders can trust release status without rechecking everything.
