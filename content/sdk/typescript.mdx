# TypeScript SDK

The ZNS TypeScript SDK provides type-safe functions for querying the registry, building memos, and validating names. It targets browsers, Node, and edge runtimes.

Source: [`sdk/typescript`](https://github.com/zcashme/zcashnames/tree/main/sdk/typescript). It's also available inside the indexer repo at [`zcashme/ZNS/sdk/typescript`](https://github.com/zcashme/ZNS/tree/master/sdk/typescript) alongside its sibling ports for Rust, Dart, Kotlin, Swift, Python, Go, and React Native.

## Installation

The SDK is distributed from the source repo. In a local project:

```sh
npm install zcashname-sdk
```

Or wire it up as a workspace package pointing at `sdk/typescript`.

## Quick start

```typescript
import { ZNS } from "zcashname-sdk";

// Create a client (defaults to testnet)
const zns = new ZNS();

// Forward resolution
const reg = await zns.resolveName("alice");

// By-address resolution
const byAddr = await zns.resolveAddress("utest1f32kn6c4…");

// Marketplace
const forSale = await zns.listings();

// Sync status
const s = await zns.status();

// Availability check
const free = await zns.isAvailable("alice");
```

The SDK defaults to `https://light.zcash.me/zns-testnet`. See [Configuration](#configuration) to override.

## Creating a Client

```typescript
import { ZNS } from "zcashname-sdk";

// Default: testnet
const zns = new ZNS();

// Explicit network
const zns = new ZNS({ network: "mainnet" });

// Custom endpoint
const zns = new ZNS({ 
  network: "testnet",
  url: "http://localhost:3000" 
});
```

## Query functions

### `resolveName(name)`

Look up a name registration.

```typescript
const reg = await zns.resolveName("alice");
// { name, address, txid, height, nonce, signature, last_action, listing }

// Returns null if name doesn't exist
const notFound = await zns.resolveName("nonexistent"); // null
```

### `resolveAddress(address, limit?, offset?)`

Look up all names registered to an address. Paginated — `limit` defaults to 50 and is capped at 500, `offset` defaults to 0.

```typescript
const registrations = await zns.resolveAddress("utest1f32kn6c4…");
// Registration[] - can be empty if no names found

// Explicit pagination
const page2 = await zns.resolveAddress("utest1f32kn6c4…", 50, 50);
```

Positional arguments — not an options object. Wire method is `resolve` with `{ query: <address>, limit, offset }`; the indexer returns `{ registrations, total }` and the SDK unwraps to the bare array.

### `listAllRegistrations(limit?, offset?)`

Every registered name, paginated. Useful for explorers, offline snapshots, and parity-checking the SDK against the raw RPC.

```typescript
// Default page (50 rows)
const rows = await zns.listAllRegistrations();

// Explicit pagination (max limit: 500)
const next = await zns.listAllRegistrations(50, 50);
```

Positional arguments. There is no separate wire method: this is `resolve` with `{ query: "", limit, offset }`. The indexer returns `{ registrations, total }`; the SDK unwraps to the bare array.

### `listings(limit?, offset?)`

All active marketplace listings, paginated. `limit` defaults to 50 and is capped at 500.

```typescript
// Get first page (50 listings by default)
const { listings, total } = await zns.listings();

// Get specific page with custom limit
const { listings, total } = await zns.listings(100, 0);

// Get next page
const { listings, total } = await zns.listings(100, 100);
```

Positional arguments — not an options object.

**Returns:** `{ listings: Listing[], total: number }`
- `listings`: Array of active marketplace listings
- `total`: Total number of listings available (use for pagination)

### `status()`

Indexer sync status.

```typescript
const s = await zns.status();
// { synced_height, admin_pubkey, uivk, address, registered, listed, pricing }
```

The field is `uivk` (Unified Incoming Viewing Key). A UIVK is enough to trial-decrypt incoming notes; a full viewing key would expose outgoing material the indexer doesn't need.

### `isAvailable(name)`

Returns `true` if the name is unregistered. Implemented as `resolveName(name) === null`.

```typescript
const free = await zns.isAvailable("alice"); // boolean
```

## Memo builders

Each protocol action has a **prepare** method that returns:
- The signing **payload** string
- A **complete** function to finalize with the signature

### CLAIM

```typescript
const prepared = zns.prepareClaim("alice", "utest1…", cost);
// { name, address, cost, payload: "CLAIM:alice:utest1…", complete }

// Sign the payload with Ed25519 (server-side), receive base64 sig
const action = prepared.complete(signature);
// { memo: "ZNS:CLAIM:alice:utest1…:<sig>", uri: "zcash:…" }

// For sovereign ownership (user-controlled), include your public key
const action = prepared.complete(signature, userPubkey);
// { memo: "ZNS:CLAIM:alice:utest1…:<sig>:<pubkey>", uri: "zcash:…" }
```

### BUY

```typescript
const prepared = zns.prepareBuy("alice", "utest1buyer…");
const action = prepared.complete(signature);
// { memo: "ZNS:BUY:alice:utest1buyer…:<sig>", uri: "zcash:…" }
```

### LIST

```typescript
const prepared = zns.prepareList("alice", 100_000_000, 1);
const action = prepared.complete(signature);
// { memo: "ZNS:LIST:alice:100000000:1:<sig>", uri: "zcash:…" }
```

### DELIST

```typescript
const prepared = zns.prepareDelist("alice", 2);
const action = prepared.complete(signature);
// { memo: "ZNS:DELIST:alice:2:<sig>", uri: "zcash:…" }
```

### UPDATE

```typescript
const prepared = zns.prepareUpdate("alice", "utest1new…", 3);
const action = prepared.complete(signature);
// { memo: "ZNS:UPDATE:alice:utest1new…:3:<sig>", uri: "zcash:…" }
```

### RELEASE

```typescript
const prepared = zns.prepareRelease("alice", 4);
const action = prepared.complete(signature);
// { memo: "ZNS:RELEASE:alice:4:<sig>", uri: "zcash:…" }
```

### SETPRICE (admin only)

```typescript
const prepared = zns.prepareSetPrice([600000000, 425000000, 300000000], 1);
const action = prepared.complete(signature);
// { memo: "ZNS:SETPRICE:3:600000000:425000000:300000000:1:<sig>", uri: "zcash:…" }
```

All `prepare*` methods validate the name via `isValidName` and throw on failure. Addresses are passed through verbatim.

## Validation

### `isValidName(name)`

```typescript
zns.isValidName("alice");       // true
zns.isValidName("bob42");       // true
zns.isValidName("my-name");     // false (hyphens not allowed)
zns.isValidName("Alice");       // false (uppercase)
```

The pattern is `/^[a-z0-9]{1,62}$/`. Matches the indexer's `validate_name`.

### `isValidUnifiedAddress(address)`

```typescript
zns.isValidUnifiedAddress("utest1f32kn6c4…");  // true
zns.isValidUnifiedAddress("u1k0evt0ahj5q…");   // true
zns.isValidUnifiedAddress("t1abc…");           // false (t-address)
```

## Pricing

### `claimCost(nameLength, pricing)`

Returns the claim cost in zatoshis from the pricing tiers.

```typescript
const status = await zns.status();
const cost = zns.claimCost(5, status.pricing!);
// 75_000_000 for a 5-letter name
```

The cost depends on the current pricing configuration from `status()`.

## ZIP-321

```typescript
const uri = zns.buildZcashUri("utest1registry…", 0.25, "ZNS:CLAIM:alice:utest1…:sig");
// "zcash:utest1registry…?amount=0.25&memo=Wk5TOkNMQUlNOmFsaWNlOnV0ZXN0MS4uLg"

const parts = zns.parseZip321Uri(uri);
// { address, amount, memoRaw, memoDecoded }
```

## UIVK Verification

```typescript
// Verify the indexer is a known ZNS instance
await zns.verify();
// Throws Error if UIVK mismatch

// Check if verified
console.log(zns.verified); // boolean
```

To skip verification (e.g., for local development):

```typescript
const zns = new ZNS({ url: "http://localhost:3000" });
// Don't call zns.verify() - just use directly
```

## Error handling

The SDK throws standard `Error` objects:

```typescript
try {
  await zns.resolveName("alice");
} catch (e) {
  if (e instanceof Error) {
    // Handle HTTP errors, invalid params, etc.
    console.error(e.message);
  }
}
```

Common error cases:
- HTTP errors from the indexer (non-2xx responses)
- Invalid name format (throws when preparing actions)
- Invalid Unified Address format
- UIVK mismatch (when calling `verify()`)

## Signature Verification

Verify registration and listing signatures independently:

```typescript
const status = await zns.status();
const adminPubkey = status.admin_pubkey;

// Verify a registration
const reg = await zns.resolveName("alice");
const isValid = await zns.verifyRegistration(reg, adminPubkey);

// Verify a listing
const listings = await zns.listings();
const listingValid = await zns.verifyListing(listings[0], adminPubkey);
```

## What's not in the SDK

The SDK does not ship OTP/HMAC helpers or admin Ed25519 signing. Memos are expected to arrive pre-signed from your server, and any local verification of the registration `signature` field is left to your Ed25519 library of choice (`@noble/ed25519` works in browsers and Node).

## Recipes

### Check availability before claiming

```typescript
async function canClaim(name: string): Promise<boolean> {
  if (!zns.isValidName(name)) return false;
  return await zns.isAvailable(name);
}
```

### Sorted marketplace listings

```typescript
const items = await zns.listings();
const sorted = items.sort((a, b) => a.price - b.price);
```

### Validate before the user submits

```typescript
function validate(name: string): string | null {
  if (!name) return "Enter a name";
  if (!zns.isValidName(name)) return "1–62 lowercase letters and digits only";
  return null;
}
```

### Complete claim flow

```typescript
// 1. Check availability
if (!(await zns.isAvailable("myname"))) {
  throw new Error("Name taken");
}

// 2. Get pricing
const status = await zns.status();
const cost = zns.claimCost(6, status.pricing!); // 6-character name

// 3. Prepare claim
const prepared = zns.prepareClaim("myname", myAddress, cost);

// 4. Sign payload (server-side with your Ed25519 key)
const signature = await signWithEd25519(prepared.payload);

// 5. Complete and get memo
const action = prepared.complete(signature);

// 6. Send Zcash transaction with action.memo to zns.registryAddress
// with at least `cost` zatoshis
```
